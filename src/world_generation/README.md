# Генерация миров и настройка поверхностей

Директория `world_generation` содержит Python-генератор Gazebo Classic world-файлов и шаблоны SDF-фрагментов для поверхностей. Генератор не является ROS-нодой: ему достаточно Python-окружения из `requirements.txt`.

## Файлы

| Файл | Назначение |
|------|------------|
| `world_generator.py` | Основной генератор миров и карт поверхностей. |
| `templates/template.world` | Шаблон одного фрагмента поверхности. В него подставляются параметры трения, контакта, отскока, цвета и геометрия. |
| `templates/world_header_template.world` | Шаблон заголовка `.world`-файла с параметрами физики Gazebo/ODE. |
| `templates/surface_example.world` | Подробный пример SDF-конфигурации мира и поверхностей. |

Сгенерированные миры сохраняются в `../worlds_library/`. Основной сценарий из `__main__` создаёт набор `../worlds_library/basis_example/<surface_name>/`, где каждый дочерний мир является бесконечной плоскостью с одной поверхностью. Рядом с каждым `.world` создаётся `<world_name>_surface_map.json`, который затем использует `surface_detector.py`.

## Запуск генератора

Из корня workspace:

```bash
source src/scripts/setup.sh
python src/world_generation/world_generator.py
```

Эта команда создаёт набор `basis_example`:

```text
../worlds_library/basis_example/
  ice/ice.world
  ice/ice_surface_map.json
  wet_asphalt/wet_asphalt.world
  wet_asphalt/wet_asphalt_surface_map.json
  ...
```

Для генератора не нужно выполнять `source devel/setup.bash`, потому что он не импортирует ROS.

## Поддерживаемые типы миров

`World` умеет генерировать:

| Метод | Что создаёт |
|-------|-------------|
| `generate_one_surface_world()` | Один бесконечный plane из выбранного материала и карту `single_surface`. |
| `generate_squares_world()` | Прямоугольную карту из квадратных плиток разных поверхностей. |
| `generate_voronoi_world()` | Карту из регионов диаграммы Вороного, сохранённых как STL-меши. |

Для `single_surface`, `squares` и `voronoi` дополнительно пишется JSON-карта поверхностей. Она нужна, чтобы ROS-нода могла определить тип поверхности под роботом без анализа геометрии Gazebo.

## Краткая шпаргалка по поверхностям

| Поверхность | `mu / mu2` | `kp`, Н/м | `kd`, Н·с/м | `soft_cfm` | `restitution` |
|-------------|------------|-----------|-------------|------------|---------------|
| Лёд | `0.03 / 0.02` | `1e7` | `1e3` | `1e-3` | `0.05` |
| Мокрый асфальт | `0.4 / 0.15` | `1e9` | `1e6` | `0.0` | `0.0` |
| Сухой бетон | `0.8 / 0.3` | `1e10` | `1e7` | `0.0` | `0.0` |
| Резина/бетон | `0.9 / 0.2` | `1e8` | `1e5` | `0.0` | `0.1` |
| Грязь | `0.6 / 0.4` | `1e6` | `1e4` | `1e-4` | `0.0` |
| Мягкий грунт | `0.5 / 0.5` | `1e5` | `1e4` | `1e-3` | `0.0` |

Ключевое правило: используйте либо пару `kp` + `kd`, либо пару `soft_cfm` + `soft_erp`, но не настраивайте обе модели мягкости одновременно. Они описывают похожую механику контакта:

```text
cfm = 1 / (kp * h + kd)
erp = kp * h / (kp * h + kd)
```

где `h` - шаг симуляции в секундах.

Когда колесо робота касается плитки, Gazebo берёт минимальные значения `mu` и `mu2` из двух контактирующих объектов: `min(mu_A, mu_B)` и `min(mu2_A, mu2_B)`.

## Трение

Документация: https://classic.gazebosim.org/tutorials?tut=friction  
SDF: `surface -> friction`

Основные параметры:

| Параметр | Рекомендуемый диапазон | Смысл |
|----------|------------------------|-------|
| `mu`, `mu2` | `0.1-1.0` | Коэффициенты трения Кулона. `mu2` часто используют как боковое трение. |
| `slip1`, `slip2` | `0.0-0.05` | Проскальзывание; большие значения дают эффект “плывущего” колеса. |
| `torsional_coefficient` | `0.0-0.05` | Коэффициент крутильного трения. |
| `torsional_slip` | `0.0-0.02` | Проскальзывание для крутильного трения. |

## Отскок

SDF: `surface -> bounce`

Основные параметры:

| Параметр | Рекомендуемый диапазон | Смысл |
|----------|------------------------|-------|
| `restitution_coefficient` | `0.0-0.4` | Коэффициент восстановления, то есть “прыгучесть” контакта. |
| `threshold` | `0.05-0.5` м/с | Скорость, ниже которой отскок подавляется. |

Для грунта и обычного движения робота обычно нужен почти нулевой отскок: `restitution_coefficient ~= 0.0-0.01`, `threshold ~= 0.1-0.2` м/с. Тогда робот не подпрыгивает при наезде.

Для жёсткого бетона или металла можно пробовать `restitution_coefficient ~= 0.2-0.4`, `threshold ~= 0.05-0.1` м/с.

## Контакт

Документация: https://classic.gazebosim.org/tutorials?tut=physics_params  
ODE Manual: https://ode.org/wiki/index.php/Manual, раздел `Soft constraint and constraint force mixing`  
SDF: `surface -> contact`

Основные параметры:

| Параметр | Диапазон | Смысл |
|----------|----------|-------|
| `kp` | `1e3 ... 1e15` Н/м | Жёсткость пружины контакта; выше = твёрже поверхность. |
| `kd` | `0.0 ... 1e13` Н·с/м | Демпфирование контакта; снижает вибрации при проникновении. |
| `soft_cfm` | `0.0 ... 1.0` | Constraint Force Mixing; `0` = твёрдый контакт, больше `0` = мягче и стабильнее. |
| `soft_erp` | `0.0 ... 1.0` | Error Reduction Parameter; доля ошибки проникновения, устраняемая за один шаг. Типовое значение `0.2`. |
| `max_vel` | `0.0 ... inf` м/с | Максимальная скорость коррекции проникновения; ограничивает “взрывные” артефакты. |
| `min_depth` | `0.0 ... ~0.01` м | Допустимая глубина проникновения до применения силы; зазор мягкости. |

Практические диапазоны, которые стоит пробовать в этом проекте:

| Параметр | Диапазон |
|----------|----------|
| `kp` | `10e5-10e7` |
| `kd` | `10-300` |
| `min_depth` | `0.01-0.05` |

Чтобы сделать грунт мягче, уменьшайте `kp` и немного увеличивайте `kd`: колёса будут глубже продавливаться до стабилизации. Слишком маленький `kp` приводит к сильному “утоплению” и нестабильности. Слишком большой `kd` может давать жёсткое затухание и колебания, если шаг интегратора и итерации solver не подобраны.

`min_depth` можно увеличить до `0.02-0.05`, чтобы получить постоянное небольшое перекрытие, визуально похожее на утопление колеса в слое грунта.

## Примеры настроек материалов

| Сценарий | Настройки |
|----------|-----------|
| Очень мягкий, вязкий грунт | Низкие `mu`, высокие `slip`, маленький `kp`, большой `kd`, нулевой `bounce`. |
| Жёсткий асфальт | Высокие `mu`, нулевой `slip`, большой `kp`, умеренный `kd`, нулевой `bounce`. |
| Жёсткий “респружинивающий” бетон | Высокие `mu`, маленький `slip`, большой `kp`, небольшой `kd`, ненулевой `bounce`. |
| Грунт/грязь | `mu = 0.2-0.4`, `slip1/slip2 = 0.05-0.2`, `kp = 1e6-1e8`, `soft_cfm = 0.001-0.01`, `restitution_coefficient = 0.0`. |
| Твёрдый грунт | `mu ~= 1.0`, `slip1/slip2 = 0.0`, `kp ~= 1e12`, `soft_cfm = 0.0`, `restitution_coefficient = 0.0`. |

Для роботов практическое правило такое: колёса на бетоне - `mu ~= 0.8`, `mu2 ~= 0.2`. Малое боковое трение полезно для дифференциального привода. Идеально скользкая поверхность - `mu = 0.0`, `mu2 = 0.0`.
